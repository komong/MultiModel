MultiModel 项目架构全景

国产多模型统一接入 · 全链路追踪 · 代码生成评测

系统定位与核心能力

MultiModel 是一套面向国产大模型的全链路开发平台，通过 LiteLLM Proxy 统一接入多个国产模型，提供 全链路追踪（Langfuse）和 代码生成评测（L1-L2 自动化）能力。

模块定位

提供三种任务调度模式与全链路追踪能力。通过 工厂模式 切换追踪后端（Console / Langfuse / Noop），支持智能路由规则引擎自动匹配最优模型。

Parallel Pipeline SmartRoute Langfuse v4 工厂模式

三种任务模式

Parallel 并行

多个模型同时处理同一输入，结果横向对比。适用于模型评测场景。

多模型 结果对比
Pipeline 流水线

按步骤串行执行，不同步骤可指定不同模型。前一步输出作为下一步输入。

串行 步骤编排
SmartRoute 智能路由

根据输入内容自动匹配最优模型。内置规则引擎（代码→DeepSeek, 翻译→MiniMax, 分析→GLM）。

模块定位

多语言代码生成评测框架，支持 L1 语法检查 和 L2 功能测试 自动化评分，涵盖 Python / JavaScript / SQL 三种语言。

L1 语法检查 L2 功能测试 Python JavaScript SQL Langfuse Dataset

评测等级体系

L1 语法检查

验证生成的代码是否能通过解释器/编译器的语法解析，返回 0.0 或 1.0。

  • Python — exec() 语法验证
  • JavaScript — new Function() 验证
  • SQL — 关键字结构检查
L2 功能测试

运行预定义的测试用例，计算通过率（0.0 ~ 1.0）。

  • Python — exec() + 测试断言
  • JavaScript — new Function() + 断言
  • SQL — 待实现
